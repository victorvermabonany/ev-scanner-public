Info

Clear math, execution rules, pros and cons, and what each free data option can realistically do.

What +EV Means

probability
  • EV compares your true win probability to the sportsbook payout.
  • If fair probability is 52% and the book price pays like 48%, the gap is value.
  • The scanner estimates true probability by removing vig from sharper reference books.
  • High EV does not mean the bet wins today; it means the price is favorable long term.

What Arbitrage Means

all outcomes
  • Arbitrage exists when the best prices across books imply less than 100% total probability.
  • You place every leg at different books and size stakes so every outcome pays back more than you risk.
  • Arbs disappear quickly, especially live, so verify every price before placing.
  • Different grading rules, voids, and fast line movement can break an apparent arb.

Pros

why it works
  • Line shopping beats taking the first price you see.
  • Kelly sizing ties risk to bankroll and edge.
  • Tracking CLV shows whether your process is beating market close.
  • Book filters keep the board focused on accounts you actually have.

Cons

where it breaks
  • Free APIs have slower refreshes, fewer requests, or limited books.
  • Props can be expensive in API calls because each event market may require separate requests.
  • Sportsbooks can limit accounts that consistently take bad prices.
  • Data delay is the biggest enemy of live betting.

Execution Checklist

before bet
  • Open the sportsbook and confirm the exact event, player, market, line, and odds.
  • Skip the play if the line moved or the market name does not match exactly.
  • For arbs, place the hardest leg first, then immediately place the other leg.
  • Log every bet so ROI and CLV are based on your actual process.

Free Feed Reality

sources
  • Demo mode is free and useful for testing the workflow.
  • PropLine is the strongest free player-prop path in this build.
  • The Odds API works well but the free tier is smaller.
  • Custom compatible endpoints let you add more free APIs when they match the expected shape.

Backend Cache

public-safe
  • Best setup for sharing the site because API keys stay off the page.
  • The backend refreshes and reuses recent responses, which protects quota.
  • It also helps with CORS when a data provider blocks browser calls.
  • Use shorter cache times for pre-match, and be careful with live betting delays.

Max Free Setup

best path
  • Use PropLine or The Odds API through backend cache mode.
  • Keep selected books tight so each scan uses fewer requests.
  • Use props only for the sports you care about because props cost more API calls.
  • Export CSV and track CLV to learn which books and markets are worth keeping.
